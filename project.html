Adversarial Training for Harmful Content Detection Neural Network

Goal:

The goal of this project was to train a neural network to detect harmful content in a 1K dataset while ensuring that the network maintains high recognition accuracy for images with AIGC-style transformations and exhibits strong robustness against adversarial samples.

Methods:
  • Analyzed and compared the accuracy of ResNet-18, ResNet-34, ResNet-50, EfficientNet, and ViT in training, selecting the optimal balance between sample size and robustness.
  • Utilized data augmentation during model training, incorporating AIGC-style images and Gaussian noise into the training set to enhance robustness.
  • Achieved a final accuracy of 92% on the native dataset, 89% on adversarial samples, and 90% on AIGC-style transformed images.

Spam Email Detection

Goal:

The goal of this project was to benchmark and compare various methods for spam email detection, evaluating accuracy, efficiency, and transferability across different techniques.

Methods:
  • Benchmarked various methods for spam detection, including traditional machine learning algorithms (Naive Bayes, KNN), deep learning models (CNN, LSTM), and large models (Zero-shot) with RAG.
  • In terms of time efficiency, traditional machine learning methods performed the best. Deep learning models achieved the highest accuracy, while large models excelled in transferability, especially when leveraging RAG for adaptation to new datasets.
  • The study concluded that while large models may excel in transferability, deep learning models can be trained to become experts on specific datasets, offering high performance on specialized tasks.
  • LSTM models showed poor performance (around 50% accuracy), suggesting that the spam classification task might be heavily dependent on keyword patterns rather than sequential dependencies.

Web Application Firewall (WAF) Based on Feature String Matching

Goal:

The goal of this project was to develop a Web Application Firewall that could efficiently intercept and filter HTTP traffic, ensuring protection against various web-based attacks while maintaining high throughput under high concurrency.

Methods:
  • Developed a Web firewall using Django, featuring a reverse proxy for HTTP message interception and a management interface for account, rule, and log management.
  • Compared rule matching performance of Hyperscan and regular expressions (re), showing Hyperscan’s superior throughput stability with large rule sets.
  • Designed an efficient rule matching engine with dynamic rule updates and real-time traffic monitoring, supporting LLM-driven automatic rule generation for easier use, and ensuring stable operation under high concurrency.

Database Management System (DBMS) with Indexing and Query Optimization

Goal:

The goal of this project was to develop a simple Database Management System (DBMS) supporting basic CRUD operations (Create, Read, Update, Delete), indexing mechanisms, and query optimization to manage and manipulate data efficiently.

Methods:
  • Developed the storage module with in-memory data indexed using a B+ tree, and used the `dill` library to serialize the B+ tree dictionary object into binary format for persistent storage on disk.
  • Implemented a buffer manager using an LRU (Least Recently Used) algorithm to efficiently manage in-memory data.
  • Applied algebraic and physical optimizations for query execution to improve the performance and efficiency of database queries.
  • Designed an execution engine that uses query trees to execute queries according to an optimized query plan.
